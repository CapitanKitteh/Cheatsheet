Podsumowanie wiedzy o GIT

    Tworzenie repozytorium na Github:
  • Wpisanie nazwy nowego repozytorium
  • Utworzenie opisu (nieobowiązkowe na tym etapie)
  • Wybór repozytorium publicznego lub prywatnego
  • Pominięcie inicjowania README.md, dodania .gitignore oraz licencji

    W folderze źródłowym wybieramy git bash here i wpisujemy komendy:
  • git init
  • git remote add origin link do repozytorium z github
  • > README.md
  • > .gitignore
  • wprowadzenie plików do gitignore jeśli jakieś są
  • git add .
  • git commit -m "Initial commit"
  • git push -u origin master (pierwszy push ustalający domyślnego brancha)
  • po zmianach add, commit, push

Informacje

Working directory/tree - pliki niedodane do śledzenia, istniejące w folderze roboczym

Index/Stage area - Dodane do śledzenia gotowe do commita

Repozytorium gita - Po commicie w folderze lokalnym

Repozytorium github - Po (commicie i) push, umieszczone na githubie

Commit message - Nie powinien zawierać więcej niż 40/60 znaków w wierszu dla ułatwienia czytania.

Pisany po angielsku w czasie present simple pierwszy wiersz.



Komendy

git config --global user.name "imię i nazwisko" - ustala dane użytkownika

git config user.name - sprawdza podane dane

git config --global user.email "adres e-mail" - ustala adresu email

git config --unset --global user.name / user.email - usuwa adres email lub dane użytkownika

git config --global core.editor - wskazuje jaki jest ustawiony domyślnie edytor

git config --global --list - lista ustawień globalnych

git config user.name "imię i nazwisko" - ustala dane użytkownika dla pojedyńczego repo (w pliku .git/config)

git XXX -h - XXX to zamiennik komendy, po wpisaniu pokaże się lista parametrów jakich można użyć z XXX

git init - inicjuje repozytorium lokalne

git remote add origin link do repozytorium z github - łączy repozytorium lokalne ze zdalnym

git status - sprawdza aktualny stan repozytorium

git diff nazwapliku.format - pokazuje porównanie pliku ze stage a zmodyfikowanym przed dodaniem aktualnej wersji do stage (podgląd przed git add .)

git diff --cached/--staged - porównanie plików ze stage do stanu plików z najnowszego commitu (podgląd zmian przed nowym commitem)

git diff hashcommitu hashdrugiegocommitu nazwapliku.format - porównuje plik względem wskazanych dwóch commitów

git add nazwapliku.format - dodaje konkretny plik do śledzenia

git add ./all - dodaje wszystkie pliki do śledzenia

git commit -m "Message commitu" - zatwierdzenie zmian w plikach do repo, aktualizowanie zmian

git commit --amend - nadpisuje ostatni commit można edytować treść i message

git commit --amend -m "Nowy message commitu" - nadpisuje tylko message commitu

git commit -a/--all - git add all git commit w jednym

git commit nazwapliku.format - commit z dodaniem zmian ze śledzonego danego pliku

git log - pokazuje wcześniejsze commity z zawartością

git log --oneline - pokazuje commity tylko z ich nazwami

git log --oneline 10 - pokazuje 10 najnowszych commitów

git log --sincee "2020" - pokazuje commity od początku 2020r

git log --since="5.5.2020" - pokazuje commity od konkretnej daty

git log --grep nazwapliku.format - wyszukuje commity dotyczące danego pliku (wielkość liter ma znaczenie)

git log --stat - pokazuje niektóre pliki i w jaki sposób zmienił je commit

git show - pokazuje co się szczegółowo zmieniło w ostatnim commicie

git show hashcommitu - pokazuje szczegółowe zmiany w danym commicie

ls / dir - pokazuje listę plików w ścieżce

dir -h - pokazuje ukryte pliki

pwd - pokazuje na jakiej ścieżce jestem

cd nazwafolderu - otwiera wskazany folder

cd nazwafolderu/nazwapliku.format - otwiera plik w nowym oknie

mkdir nazwafolderu - tworzy nowy folder

touch/echo/>>/> nazwapliku.format - tworzy nowy plik w ścieżce

rm nazwapliku.format - usuwa plik z folderu roboczego

git rm --cached nazwapliku.format - usuwa ze stage (plik istnieje w folderze roboczyma jego stan to nieśledzony)

git rm nazwapliku.format - usuwa plik z folderu roboczego i stage (kolejny commit to uwzględni tak samo status, plik nadal będzie w historii repozytorium)

git reset HEAD / nazwapliku.format - usuwa pliki(HEAD) lub wskazany plik ze stage (istnieją w folderze i są nadal śledzone)

git mv nazwapliku.format nowanazwapliku.format - zmienia nazwę pliku

git cat-file -p nazwapliku.format - wyświetli zawartość pliku

git cat-file -t nazwapliku.format - wyświetli typ pliku

git checkout --nazwapliku.format - przywraca wersję zmodyfikowanego pliku do wersji w stage (jeśli commit jeszcze nie był wykonany)

git checkout HEAD --nazwapliku.format - przywraca wersję pliku z najnowszego commita

git checkout hashcommita - przywraca stage do stanu przed wskazanym commitem

git hash-object nazwapliku.format - podaje hash (identyfikator pliku w folderze lokalnym, oznaczonym jako 2losowe znaki i 38 kolejnych w nazwie pliku) można używać też pierwszych 7 początkowych znaków

code - otwiera edytor

git clone nazwafolderuzrepo nazwaklonowanegofolderu - klonuje folder z repozytorium

git clone link do klonowania z github - klonuje repozytorium z github

git checkout -b nazwabrancha - tworzy nową gałąź repozytorium

git branch -a - pokazuje listę utworzonych gałęzi

git checkout master - przenosi na wskazaną gałąź

git branch -d nazwabrancha - usuwa wskazaną galąź

git merge nazwabrancha - łączy gałąź wskazaną z tą, na której się znajdujemy

q - zakończenie komendy przewijanej (najczęściej listy)

git push - przesyła zmiany z repo lokalnego na zdalne (github)

git pull - pobiera zmiany z repo zdalnego na lokalnego

git remote -v - wskazuje do jakich miejsc jest przypisany push i pull
