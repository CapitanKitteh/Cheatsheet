Size and units

Height - wysokość i width - szerokość określa się za pomocą:
            auto - domyślne, 
            Nieresponsywne (nie dostosowujące się do rozmiaru okna przeglądarki):
            cm, mm, in (=96px/2,54cm), pt (point=1/72in), pc (picas 1=12pt)
            Responsywne:
            em (current font size, default is 16px which is 1em)
            vw relative to 1% of viewport width
            vh relative to 1% of viewport height 
            vmin relative to 1% viewport smaller dimension
            vmax relative to 1% viewport larger dimension 
            % relative to parent element 
            inherit
            Viewport to rozmiar okna przeglądarki, jeśli ma 50cm szerokości jego 1vw=0.5cm


Atrybuty selektorów

W ten sposób można okreslić wygląd wielu, specyficznie określonych elementów.
1. Przykład dotyczący każdego linku z dowolnie określonym targetem: 

          a[target] {
            background-color: #000283;
          }
2. Przykład dotyczący tylko linków z target="_blank": 

          a[target~="_blank"] {
            background-color: #000283;
          }
3. Przykład dotyczący tylko tytułów w których treści znajdowało sie słowo "easy": 

          [title~="easy"] {
            background-color: #000283;
          }
4. Przykład dotyczący każdego id w spisie treści zaczynającego się na "border": Może to być całe słowo, lub łączone "-" 

          #tableofcontents[id|=border] {
            background-color: #000283;
          }
5. Przykład dotyczący każdego id w spisie treści zaczynającego się na "bord": Nie musi to byc całe słowo 

          #tableofcontents[id|bord] {
            background-color: #000283;
          }
6. Przykład dotyczący każdej classy kończącej się na "preview" : Nie musi to być całe słowo. 

          [class$="preview"] {
            background-color: #000283;
          }
7. Przykład dotyczący każdego list element zawierającego w class "shop" : Nie musi być całym słowem i nie musi być początkiem ani końcem słowa, np. będzie dotyczył shoplist, listofshops, shopmap itd. 

          li[class*="shop"] {
            background-color: #000283;
          }

Kolory

Kolory można określać przy pomocy wartości rgb, hex, hsl a również rgba i hsla
Dla wszystkich przykładów został użyty ten sam kolor, w przypadku rgba i hsla transparentny w 50%.
1. Kolory określane za pomocą wartości rgb (red, green, blue) 0-255: 

            p {
              background color: rgb(0, 2, 131) 
            }
2. Kolory określane za pomocą wartości rgba (red, green, blue, alpha) 0-255, alpha 0.0 - transparentny 1.0 - pełny kolor : 

            p {
              background-color: rgba(0, 2, 131, 0.5)
            }
3. Kolory określane za pomocą wartości hex (#000000-#ffffff w czym 2 pierwsze to red, dwa kolejne green a dwa ostatnie blue [#rrggbb]) : 

            p {
              background-color: #000283;
            }
Kolor czarny: #000000, kolor biały: #ffffff, kolor szary i jego odcienie są często tworzone z powtarzających się wartości, np. #c4c4c4 czy #1f1f1f
4. Kolory określane za pomocą wartości hsl (hue, saturation, lightness) Hue - odcień/kolor 0-360 przy czym 0 - czerwony 120 - zielony a 240 - niebieski
Saturation - skala szarości 0% - odcien szary, 100% to pełny kolor
Lightness - skala jasności koloru 0% - czarny, 25% - ciemny odcień danego koloru, 50% - pełny kolor, 75% - jasny odcień danego koloru, 100% biel 

            p {
              background-color: hsl(239, 100%, 26%);
            }
5. Kolory określane za pomocą wartości hsla (hue, saturation, lightness, alpha) alpha 0.0 - transparentny 1.0 - pełny kolor : 

            p {
              background-color: hsla(239, 100%, 26%, 0.5)
            }


Gradient colors

Przejścia kolorów mogą być liniowe (w dół, w górę, w prawo, w lewo, po przekątnej, wedle stopnia) jak i promieniowe określane od ich środka.
1. Aby utworzyć liniowe przejście potrzebne są minimum dwa kolory
Syntax: 

            p {
            background-image: linear-gradient(direction, color1, color2);
            }
1.1 Przykład zielono-niebieskiego przejścia top to bottom (default) : 

            p {
              background-image: linear-gradient(green, blue);
            }
Zamiana kolorów miejscami pozwoli uzyskać efekt bottom to top.
1.2 Przykład zielono-niebieskiego przejścia w prawo : 

            p {
              background-image: linear-gradient(to right, green, blue);
            }
1.3 Przykład zielono-niebieskiego przejścia w lewo : 

            p {
              background-image: linear-gradient(to left, green, blue);
            }
1.4 Przykład zielono-niebieskiego przejścia po przekątnej do górnej lewej strony : 

            p {
              background-image: linear-gradient(to top left, green, blue);
            }
1.5 Przykład zielono-niebieskiego przejścia po przekątnej do górnej prawej strony : 

            p {
              background-image: linear-gradient(to top right, green, blue);
            }
1.6 Przykład zielono-niebieskiego przejścia po przekątnej do dolnej lewej strony : 

            p {
              background-image: linear-gradient(to bottom left, green, blue);
            }
1.7 Przykład zielono-niebieskiego przejścia po przekątnej do dolnej prawej strony : 

            p {
              background-image: linear-gradient(to bottom right, green, blue);
            }
1.8 Określanie procentowe ilości koloru :
Wystarczy określić jeden lub pożądane wartości, w tym przykładzie red zajmie domyślnie pozostałe 50%. 

            p {
              background-image: linear-gradient(coral 10%, orange 40%, red);
            }
1.9 Więcej kolorów : 

            p {
              background-image: linear-gradient(grey, teal 30%, coral, black, gold)
            }
1.10 Określanie kąta przejścia :
Takie samo jak w przypadku to top czyli 0deg oraz 360deg (red on bottom)
Tak samo jak w przypadku to right, czyli 90deg
Tak samo jak w przypadku to bottom (domyślnie), czyli 180deg (red on top) w przykładzie poniżej
Tak samo jak w przypadku to left, czyli 270deg

            p.test {
              background-image: linear-gradient(180deg, #fa7272 30%, #ecff83 60%, #6890ff 100%); 
            }
1.11 Brak przejścia, pasy kolorów, czyli wielokrotnie określanie umiejscowienia koloru : 

            p {
              background-image: linear-gradient(red 20%, gray 20% 40%, green 40% 60%, black 60% 80%, blue 80%);
            }
1.12 Powtarzanie określonego przejścia :
0% czerwonego oznacza jego początek tuż przy krawędzi w tym przypadku lewej ponieważ kierunek określony jest w prawo
20% żółtego oznacza jego położenie na 20% ze 100%
30% przy kolorze niebieskim oznacza miejsce jego zakończenia
W przypadku czerwonego 0%, żółtego 60% i niebieskiego 40% żółty będzie położony w miejscu 60% ze 100% a niebieski zniknie z przejścia.

            p {
              background-color: repeating-linear-gradient(#fa7272 0%, #ecff83 20%, #6890ff 30%)
            }
2. Przejścia promieniowe również są tworzone z minimum dwóch kolorów
Syntax: p { background-image: radial-gradient(shape size at position, color1, color2); }
2.1 Kształt circle albo ellipse(default) 

            p {
              background-image: radial-gradient(circle, green, blue);
            }
            p {
              background-image: radial-gradient(ellipse, green, blue);
            }
2.2 Umieszczenie środka gradientu i rozmiar do wszystkich pierwsza wartość odnosi się do osi x/położenia poziomego a druga do osi y/położenia pionowego
Closest-side i closest-corner są mniejsze a farthest-side i farthest-corner są większe 

            p {
              background-image: radial-gradient(closest-side 20% 70%, green, blue);
            }
2.3 Powtarzanie gradientu :
0% zielonego w centrum
45% niebieskiego umieszczone na długości 100% promienia nie całego bloku
75% koloru teal jest wyznacznikiem końca przejścia i tam zaczyna się powtarzanie 

            p {
              background-image: repeating-radial-gradient(circle, green 0%, blue 45%, teal 75%);
            }
