Size and units

Height - wysokość i width - szerokość określa się za pomocą:
            auto - domyślne, 
            Nieresponsywne (nie dostosowujące się do rozmiaru okna przeglądarki):
            cm, mm, in (=96px/2,54cm), pt (point=1/72in), pc (picas 1=12pt)
            Responsywne:
            em (current font size, default is 16px which is 1em)
            vw relative to 1% of viewport width
            vh relative to 1% of viewport height 
            vmin relative to 1% viewport smaller dimension
            vmax relative to 1% viewport larger dimension 
            % relative to parent element 
            inherit
            Viewport to rozmiar okna przeglądarki, jeśli ma 100cm szerokości jego 1vw=1cm


Selector attributes

W ten sposób można okreslić wygląd wielu, specyficznie określonych elementów.
1. Przykład dotyczący każdego linku z dowolnie określonym targetem: 

          a[target] {
            background-color: #000283;
          }
2. Przykład dotyczący tylko linków z target="_blank": 

          a[target~="_blank"] {
            background-color: #000283;
          }
3. Przykład dotyczący tylko tytułów w których treści znajdowało sie słowo "easy": 

          [title~="easy"] {
            background-color: #000283;
          }
4. Przykład dotyczący każdego id w spisie treści zaczynającego się na "border": Może to być całe słowo, lub łączone "-" 

          #tableofcontents[id|=border] {
            background-color: #000283;
          }
5. Przykład dotyczący każdego id w spisie treści zaczynającego się na "bord": Nie musi to byc całe słowo 

          #tableofcontents[id|bord] {
            background-color: #000283;
          }
6. Przykład dotyczący każdej classy kończącej się na "preview" : Nie musi to być całe słowo. 

          [class$="preview"] {
            background-color: #000283;
          }
7. Przykład dotyczący każdego list element zawierającego w class "shop" : Nie musi być całym słowem i nie musi być początkiem ani końcem słowa, np. będzie dotyczył shoplist, listofshops, shopmap itd. 

          li[class*="shop"] {
            background-color: #000283;
          }


Combinators

Są to łączniki wskazujące na relację między selektorami
Przykłady:
1. Każdy span w class="quots" :
Każdy inny span poza określoną klasą nie będzie brany pod uwagę 

            .quots span {
              color: #00283;
            }
2. Jedynie li bezpośrednio znajdujące się po ol będą ostylowane, w tym przypadku listy w liście, durga lista ul również posiadająca li nie będzie brana pod uwagę : 

            ol > li {
              color: #000283;
            }
3. Tylko element w tym przypadku div tuż po h3 będzie ostylowany, jeśli coś znajduje się pomiędzy nimi, reguła przestanie działać : 

            h3.testy + div {
              color: #000283;
            }
4. Przykład taki sam jednak łącznik będzie dotyczył każdego div niezależnie od tego, czy coś się znajduje między nim a h3 : 

            h3.testy ~ div {
              color: #000283;
            }


Colors

Kolory można określać przy pomocy wartości rgb, hex, hsl a również rgba i hsla
Dla wszystkich przykładów został użyty ten sam kolor, w przypadku rgba i hsla transparentny w 50%.
1. Kolory określane za pomocą wartości rgb (red, green, blue) 0-255: 

            p {
              background color: rgb(0, 2, 131) 
            }
2. Kolory określane za pomocą wartości rgba (red, green, blue, alpha) 0-255, alpha 0.0 - transparentny 1.0 - pełny kolor : 

            p {
              background-color: rgba(0, 2, 131, 0.5)
            }
3. Kolory określane za pomocą wartości hex (#000000-#ffffff w czym 2 pierwsze to red, dwa kolejne green a dwa ostatnie blue [#rrggbb]) : 

            p {
              background-color: #000283;
            }
Kolor czarny: #000000, kolor biały: #ffffff, kolor szary i jego odcienie są często tworzone z powtarzających się wartości, np. #c4c4c4 czy #1f1f1f
4. Kolory określane za pomocą wartości hsl (hue, saturation, lightness) Hue - odcień/kolor 0-360 przy czym 0 - czerwony 120 - zielony a 240 - niebieski
Saturation - skala szarości 0% - odcien szary, 100% to pełny kolor
Lightness - skala jasności koloru 0% - czarny, 25% - ciemny odcień danego koloru, 50% - pełny kolor, 75% - jasny odcień danego koloru, 100% biel 

            p {
              background-color: hsl(239, 100%, 26%);
            }
5. Kolory określane za pomocą wartości hsla (hue, saturation, lightness, alpha) alpha 0.0 - transparentny 1.0 - pełny kolor : 

            p {
              background-color: hsla(239, 100%, 26%, 0.5)
            }


Gradient colors

Przejścia kolorów mogą być liniowe (w dół, w górę, w prawo, w lewo, po przekątnej, wedle stopnia) jak i promieniowe określane od ich środka.
1. Aby utworzyć liniowe przejście potrzebne są minimum dwa kolory
Syntax: 

            p {
            background-image: linear-gradient(direction, color1, color2);
            }
1.1 Przykład zielono-niebieskiego przejścia top to bottom (default) : 

            p {
              background-image: linear-gradient(green, blue);
            }
Zamiana kolorów miejscami pozwoli uzyskać efekt bottom to top.
1.2 Przykład zielono-niebieskiego przejścia w prawo : 

            p {
              background-image: linear-gradient(to right, green, blue);
            }
1.3 Przykład zielono-niebieskiego przejścia w lewo : 

            p {
              background-image: linear-gradient(to left, green, blue);
            }
1.4 Przykład zielono-niebieskiego przejścia po przekątnej do górnej lewej strony : 

            p {
              background-image: linear-gradient(to top left, green, blue);
            }
1.5 Przykład zielono-niebieskiego przejścia po przekątnej do górnej prawej strony : 

            p {
              background-image: linear-gradient(to top right, green, blue);
            }
1.6 Przykład zielono-niebieskiego przejścia po przekątnej do dolnej lewej strony : 

            p {
              background-image: linear-gradient(to bottom left, green, blue);
            }
1.7 Przykład zielono-niebieskiego przejścia po przekątnej do dolnej prawej strony : 

            p {
              background-image: linear-gradient(to bottom right, green, blue);
            }
1.8 Określanie procentowe ilości koloru :
Wystarczy określić jeden lub pożądane wartości, w tym przykładzie red zajmie domyślnie pozostałe 50%. 

            p {
              background-image: linear-gradient(coral 10%, orange 40%, red);
            }
1.9 Więcej kolorów : 

            p {
              background-image: linear-gradient(grey, teal 30%, coral, black, gold)
            }
1.10 Określanie kąta przejścia :
Takie samo jak w przypadku to top czyli 0deg oraz 360deg (red on bottom)
Tak samo jak w przypadku to right, czyli 90deg
Tak samo jak w przypadku to bottom (domyślnie), czyli 180deg (red on top) w przykładzie poniżej
Tak samo jak w przypadku to left, czyli 270deg

            p.test {
              background-image: linear-gradient(180deg, #fa7272 30%, #ecff83 60%, #6890ff 100%); 
            }
1.11 Brak przejścia, pasy kolorów, czyli wielokrotnie określanie umiejscowienia koloru : 

            p {
              background-image: linear-gradient(red 20%, gray 20% 40%, green 40% 60%, black 60% 80%, blue 80%);
            }
1.12 Powtarzanie określonego przejścia :
0% czerwonego oznacza jego początek tuż przy krawędzi w tym przypadku lewej ponieważ kierunek określony jest w prawo
20% żółtego oznacza jego położenie na 20% ze 100%
30% przy kolorze niebieskim oznacza miejsce jego zakończenia
W przypadku czerwonego 0%, żółtego 60% i niebieskiego 40% żółty będzie położony w miejscu 60% ze 100% a niebieski zniknie z przejścia.

            p {
              background-color: repeating-linear-gradient(#fa7272 0%, #ecff83 20%, #6890ff 30%)
            }
2. Przejścia promieniowe również są tworzone z minimum dwóch kolorów
Syntax: p { background-image: radial-gradient(shape size at position, color1, color2); }
2.1 Kształt circle albo ellipse(default) 

            p {
              background-image: radial-gradient(circle, green, blue);
            }
            p {
              background-image: radial-gradient(ellipse, green, blue);
            }
2.2 Umieszczenie środka gradientu i rozmiar do wszystkich pierwsza wartość odnosi się do osi x/położenia poziomego a druga do osi y/położenia pionowego
Closest-side i closest-corner są mniejsze a farthest-side i farthest-corner są większe 

            p {
              background-image: radial-gradient(closest-side 20% 70%, green, blue);
            }
2.3 Powtarzanie gradientu :
0% zielonego w centrum
45% niebieskiego umieszczone na długości 100% promienia nie całego bloku
75% koloru teal jest wyznacznikiem końca przejścia i tam zaczyna się powtarzanie 

            p {
              background-image: repeating-radial-gradient(circle, green 0%, blue 45%, teal 75%);
            }


Text fonts

Dołączamy czcionki np. z Google Fonts: Po wybraniu czcionki zaznaczam style, które mnie intesują, następnie jeśli będę umieszczać reguły css w pliku html wybieram link (umieszczam go w head),
jeśli będę umieszczać wszystko w osobnym pliku css wybieram @import (umieszczam go na początku pliku css), dodatkowo ukazuje się informacja jak opisać czcionkę.
    Bezpieczna lista (interpretowanych bezproblemowo w większości przeglądarek):
  • Arial
  • Arial Black
  • Comic Sans Ms
  • Courier New
  • Georgia
  • Impact
  • Times New Roman (default)
  • Trebuchet Ms
  • Verdana
Jeśli nazwa jest wieloczłonowa piszemy ją w cudzysłowie, "Times New Roman", a jeśli podajemy kilka nazw wygląda to tak: "Lucida Console", Courier, monospace
font-weight: normal (400), bold (700), bolder, lighter, initial (to it's default), inherit, 100-900
font-style: normal, italic (kursywa) lub oblique (wymuszona kursywa, nie zawsze wspierana)
font-variant: normal (default), small-caps (małe litery będą powiększone, ale będą mniejsze niż duże litery), initial, inherit
font-size: określane w px, %, em, za pomocą viewport, rownież określanie przez small, medium, large, smaller, larger, x-small, xx-small, x-large, xx-large, initial, inherit.
Default to 16px=1em.
Określanie % względem px: 

            p {
              font-size: 30px;
              line-height: 140%
            }
            140% z 30px da rozmiar 42px
Shortcut: 

            p. {
              font: italic small-caps bold 30px Arial, serif;
            }


Actions

Przykład użycia akcji na odnośnikach: a:link - normalny, nieodwiedzony a:hover - najechany a:active - kliknięty (i ewentualnie przytrzymany) a:visited - użyty / odwiedzony Gdy ustalamy wiele lub wszystkie obowiązuje kolejność: link, visited, hover, active. Efekty interakcji: Po kliknięciu/przytrzymaniu elementu o class: pic jego transparentność się zmniejszy 

            .pic:active {
              opacity: .4;
            }
Po najechaniu na element (dobrze wygląda ze zdjęciami) o class: .pic utworzy się box shadow dając efekt uniesienia elementu, można to zrobić również z text-shadow 

            .pic:hover {
              box-shadow 5px 5px 20px rgba(0, 0, 0, .7);
            }
Płynne przejścia opisywane są w tym przykładzie jako 1sekunda 

            p.test:hover {
            transition: 1s;
            }
Opóźnienie płynnego przejścia 

            p.test2:hover {
              transition-delay: 3s;
            }

Text

1. text-align: left, right, center, justify, initial, inherit
1.1 vertical-align: baseline (default), np. 20px/em/% (w górę) lub 20px/em/% (w dół), sub, super, top, middle, bottom, initial, inherit
2. text-decoration: none, initial, inherit
2.1 text-decoration-line: none, underline, overline, line-through, initial, inherit
2.2 text-decoration-style: solid, double, dotted, dashed, wavy, initial, inherit
2.3 text-decoration-color: red, #000283 itd.
Przykład przekreślenia w stylu dashed w kolorze czerwonym: 

            p {
              line-decoration: like-through dashed red;
            }

            Można to napisać również w ten sposób:

            p {
              line-decoration-line: line-through;
              line-decoration-style: dashed;
              line-decoration-color: red;
            }
3. text-indent: 20px - wcięcie tekstu/odstęp akapitowy na 20px, określać go można również w % lub em
Wartości minusowe poskutkują wcięciem w prawo chowając tekst
Przykład wcięcia w em: 

            p {
              text-indent: 4em;
            }
4. letter-spacing: np. 2px, 20px, -2px - odległość liter od siebie
4.1 word-spacing: np. 2px, 20px, -40px - odległość wyrazów od siebie
5. text-transform: none (default), capitalize, lowercase, uppercase, initial, inherit
5.1 capitalize - pierwsza litera w każdym słowie będzie duża
5.2 lowercase - każda litera będzie mała
5.3 uppercase - każda litera będzie duża
6. white-space: normal, nowrap, pre, pre-line, pre-wrap, initial, inherit
6.1 normal (default) 

            p {
              white-space: normal;
            }
6.2 nowrap prezentuje tekst o takiej długości w jakiej jest zapisany pierwotnie 

            p {
              white-space: nowrap;
            }
6.3 pre prezentuje tekst dokładnie tak samo jak został napisany 

            p {
              white-space: pre;
            }
6.4 pre-line prezentuje tekst tak samo jak został napisany przyrównując każdą następną linijkę do krawędzi 

            p {
              white-space: pre-line;
            }
6.5 pre-wrap prezentuje tekst zachowując go tak jak został napisany, niemniej to co jest zbyt długie umieszcza w lini poniżej 

            p {
              white-space: pre-wrap;
            }
7. line-height: normal (default), number, px/em/cm/%, initial, inherit
Dozwolone są wartości ujemne
7.1 number zalecane, ustala odtęp od każdej lini na podstawie jej wysokości 

            p {
              line-height: 0.5;
            }
            Domyślnie:
            p {
              line-height: 1.0;
            }
            p {
              line-height: 7.0;
            }
7.2 Jednostki 

            p {
              line-height: -3px;
            }
            p {
              line-height: 2em;
            }
            p {
              line-height: 30%;
            }
            100% i 1em to normal
8. direction: ltr (default), rtl, initial, inherit - określa do której krawędzi tekst jest dorównany
8.1 left to right - przyrównanie do lewej krawędzi 

            p {
              direction: ltr;
            }
8.2 right to left - przyrównanie do prawej krawędzi 

            p {
              direction: rtl;
            }
9. text-overflow: clip, ellipsis, string, initial, inherit - określa co się dzieje z tekstem wychodzącym poza widoczne pole
W przykładach overflow: hidden ukrywa wszystko co wychodzi poza okno przeglądarki
9.1 clip ucina tekst 

            p {
              white-space: nowrap;
              overflow: hidden;
              text-overflow: clip;
            }
9.2 ellipsis ucina tekst zostawiając (...) 

            p {
              white-space: nowrap;
              overflow: hidden;
              text-overflow: ellipsis;
            }
10 text-shadow: none, h-shadow, v-shadow, blur-radius, color, initial, inherit
10.1 Cień można określić w następujący sposób: Pierwsza wartość to przesunięcie cienia względem tekstu w poziomie w przykładzie wynosi 2px
Druga wartość to przesunięcie w pionie również 2px
Trzecia wartość to kolor w tym przypadku #000283
Przykład średnio czytelny 

            p {
              text-shadow: 2px 2px #000283;
            }
            Dodam trzecią wartość, czyli blur-radius o wartości 5px 
            p {
              text-shadow: 2px 2px 5px #000283;
            }
10.2 Przykład na białym kolorze tekstu: 

            p {
              color: white;
              text-shadow: 2px 2px 5px #000000
            }
10.3 Przykład efektu neon w kolorze mięty: 

            p {
              text-shadow: 0px 0px 5px #03f7f7;
            }
10.4 Podwójny kolor neonu: 

            p {
              text-shadow: 0px 0px 5px #ff643d, 0px 0px 5px #8671ff; 
            }
10.5 Dopuszczalne są także wartości ujemne: 

            p {
              text-shadow: -5px -5px 5px #000283;
            }

Position

Określanie pozycji elementów można wykonać za pomocą
position: static, relative, absolute, sticky, fixed oraz określena ich pozycji względem top, bottom, right, left.
1. Static - default dla elementów html
2. Relative
Rezerwuje sobie miejsce z którego został przesunięty i przesuwamy go wedle jego własnej pozycji

            div.prel {
              position: relative;
              left: 200px;
              border: 2px solid #000283;
              background-color: #000283;
              color: white;
            }
3. Absolute
Ustawiamy go względem położonego wyżej elementu z określoną pozycją relatywną jeśli takiej nie ma uznaje zamiast niej body, nie rezerwuje sobie miejsca
